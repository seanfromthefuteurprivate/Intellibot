                              EXAMPLE TRADE FLOW
═══════════════════════════════════════════════════════════════════════════════

SCENARIO: SPY 450 CALL @ $1.50 (0DTE)

Step 1: HYDRA Intelligence
───────────────────────────
HYDRA says:
  direction: BULLISH
  regime: TRENDING_UP
  blowup_probability: 45%
  gex_regime: NEGATIVE (trending)
  gex_flip_distance_pct: 2.5%
  flow_bias: AGGRESSIVELY_BULLISH
  flow_sweep_direction: CALL_HEAVY

Step 2: Gate Validation
───────────────────────
  ✅ GATE 1 (Direction):  BULLISH allows CALLS
  ✅ GATE 2 (Blowup):     45% < 70% → PASS (full size)
  ✅ GATE 3 (Regime):     TRENDING_UP → Tradeable
  ✅ GATE 4 (GEX Flip):   2.5% > 1% → Safe zone

Step 3: Confidence Scoring
──────────────────────────
  Base:                        60
  + Aggressive flow:          +15
  + Negative GEX:             +10
  + Call sweeps aligned:       +5
  + Momentum uptrend:         +15
  ────────────────────────────────
  Final Confidence:            90%

Step 4: Trade Generation
────────────────────────
  ✅ Liquidity:  Bid-ask spread 5%, mid $1.48
  ✅ Affordability: $150 cost < $250 cap
  ✅ Diversity: First SPY CALL today
  ✅ Cooldown: No recent SPY CALL
  ✅ Deduplication: Not seen before

  → Generate BUY call

Step 5: Alpaca Execution
────────────────────────
  Log HYDRA context:
    ALPACA_HYDRA_CONTEXT: dir=BULLISH regime=TRENDING_UP blowup=45% flow=AGGRESSIVELY_BULLISH

  Calculate size:
    _get_hydra_size_multiplier() = 1.0x (blowup 45% < 50%)
    Position size: 1 contract ($150)

  Execute:
    Market BUY 1x SPY 450C @ $1.50
    Entry logged, position tracked

Step 6: Position Monitoring
───────────────────────────
  Current price: $1.88 (+25%)
  Exit condition: TP1 hit (+25%)

  Execute exit:
    Market SELL 1x SPY 450C @ $1.88
    PnL: +$38 (+25.3%)
    Record to database

═══════════════════════════════════════════════════════════════════════════════
                          FALLBACK & ERROR HANDLING
═══════════════════════════════════════════════════════════════════════════════

HYDRA Disconnected
──────────────────
  Symptom: hydra.connected = False
  Action:  CPL falls back to momentum-only validation
  Impact:  Gates return PASS with reduced confidence (50%)
  Log:     "HYDRA_DISCONNECTED: Trading may be limited"

HYDRA Exception
───────────────
  Symptom: get_hydra_intel() throws error
  Action:  Catch exception, log warning, fall back to momentum
  Impact:  Trading continues in degraded mode
  Log:     "HYDRA entry check failed {ticker}: {exception}"

Momentum Data Missing
─────────────────────
  Symptom: No 5-min bars available
  Action:  Use HYDRA confidence alone (no momentum boost/penalty)
  Impact:  Slightly lower confidence (60-80 range vs 75-90)
  Log:     "HYDRA_APPROVED: {direction} {regime} (no momentum data)"
